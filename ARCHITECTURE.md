# Claude Profiles — Architecture

## Overview

Claude Profiles lets multiple Claude Desktop and Claude Code instances run simultaneously, each with a fully isolated account. This is useful for separating work, personal, and other Claude accounts on the same Mac.

## Components

```
claude-profiles/
├── claude_profiles/        # Core Python library + CLI
│   ├── cli.py              # argparse entry point (claude-profiles <cmd>)
│   ├── desktop.py          # Claude Desktop bundle management
│   ├── code.py             # Claude Code (env-var) isolation
│   ├── profile.py          # Profile data model + paths
│   └── manager.py          # Profile persistence (profiles.json)
├── menubar/
│   └── menubar.py          # rumps menu bar app (py2app entry point)
├── setup_app.py            # py2app build config → dist/Claude Profiles.app
├── Makefile                # make install-app: build + copy + launch
└── resources/
    └── icon.icns           # Generated by scripts/make_icon.py
```

## Isolation Strategy

### Claude Desktop

**Goal**: each profile is a completely separate Claude account, with no shared session state.

**Mechanism**:
1. Copy `/Applications/Claude.app` → `~/Applications/Claude-{slug}.app`
2. Patch the outer bundle's `Info.plist`:
   - `CFBundleIdentifier` → `com.anthropic.claude.profile.{slug}` (Keychain isolation)
   - `CFBundleName` → `Claude-{slug}` (Electron derives helper paths + userData from this)
   - `CFBundleDisplayName` → profile display name (what the Dock shows)
3. Rename all Electron helper bundles inside `Contents/Frameworks/` to match:
   - `Claude Helper.app` → `Claude-{slug} Helper.app`
   - `Claude Helper (GPU).app` → `Claude-{slug} Helper (GPU).app` … etc.
   - Also renames the binary inside each helper and patches its `Info.plist`
4. Ad-hoc re-sign the whole bundle inside-out (nested helpers first, outer app last).  
   `--deep` alone is insufficient for Electron — nested `.app` bundles inside `Frameworks/` must be signed individually.
5. Clear Gatekeeper quarantine with `find ... | xargs -0 xattr -c`
6. Launch via `open -n <app> --args --user-data-dir=~/Library/Application\ Support/Claude-{slug}`

**Why steps 3 + 6 are both needed**:
- Step 3 (renaming helpers): required for the app to launch at all. Electron constructs the helper path as `Frameworks/{CFBundleName} Helper.app`. If `CFBundleName` changes but helpers aren't renamed, Electron panics with "Unable to find helper app".
- Step 6 (`--user-data-dir`): required for account isolation. Claude Desktop hardcodes its data path regardless of `CFBundleName`; passing the flag at launch forces Electron to use a profile-specific directory (`~/Library/Application Support/Claude-{slug}`), giving each profile its own cookies, localStorage, and session storage.

**Keychain isolation** is handled automatically by `CFBundleIdentifier`: macOS scopes Keychain entries to the bundle identifier, so credentials are never shared across profiles.

### Claude Code

Claude Code reads `CLAUDE_CONFIG_DIR` at startup to locate its config. We set:

```
CLAUDE_CONFIG_DIR=~/.claude-profiles/{slug}/claude-code/
```

before launching the terminal. Each profile gets its own `CLAUDE.md`, projects state, and settings.

The terminal is Ghostty (if installed) or Terminal.app as fallback:
```bash
ghostty -e bash -lc "CLAUDE_CONFIG_DIR=... claude; exec bash"
```

## Menu Bar App

`menubar/menubar.py` is a `rumps`-based PyObjC app that lives in the macOS menu bar. It:
- Reads `~/.claude-profiles/profiles.json` for the profile list
- Spawns `claude-profiles launch {name}` / `claude-profiles code {name}` in subprocesses
- Checks for Claude Desktop updates (compares synced version in `claude-version.json` vs installed)
- Runs an update sync in a background thread to avoid blocking the main run loop

Built with py2app into `dist/Claude Profiles.app` — a fully standalone `.app` bundle that includes Python, all dependencies, and the `claude-profiles` CLI script.

## Build + Install

```
make install-app
```

1. `py2app` bundles `menubar/menubar.py` + all deps into `dist/Claude Profiles.app`
2. Copies to `/Applications/Claude Profiles.app`
3. Clears quarantine on the bundle
4. Launches the app

The bundled app includes:
- `Contents/MacOS/Claude Profiles` — Mach-O launcher
- `Contents/MacOS/python` — bundled CPython 3.11
- `Contents/Resources/menubar.py` — the menu bar app script
- `Contents/Resources/bin/claude-profiles` — CLI shim
- `Contents/Resources/claude_profiles/` — the Python library

## Profile Data

All profile state is stored under `~/.claude-profiles/`:

```
~/.claude-profiles/
├── profiles.json              # list of profiles (name, display_name, emoji)
├── claude-version.json        # last synced Claude Desktop version
└── {slug}/
    └── claude-code/           # CLAUDE_CONFIG_DIR for this profile's Claude Code
~/Applications/
└── Claude-{slug}.app          # isolated Claude Desktop bundle per profile
~/Library/Application Support/
└── Claude-{slug}/             # Electron userData per profile (session, cookies, etc.)
```

## Known Limitations / Future Work

- **Python/conda dependency**: the menu bar app requires a conda environment to build. A better distribution would be a fully standalone binary (e.g., using PyInstaller `--onefile` or rewriting the menu bar in Swift).
- **Other apps**: the profile isolation technique could be extended to other Electron-based apps like OpenAI Codex Desktop or Cursor, using the same bundle-copy + helper-rename + userData-dir approach.
- **Auto-update**: when Claude Desktop updates, profile bundles must be re-synced via `claude-profiles sync`. This could be automated with a LaunchAgent.
- **First-launch Gatekeeper dialog**: even with quarantine cleared and ad-hoc signing, macOS may show a one-time "app from unidentified developer" dialog on first launch of each profile bundle. This is expected and can be dismissed once.
